🔄 四種 AI 轉換模式

🎨

風格轉換

保留圖片的構圖與主體內容，套用全新的藝術畫風，如水彩、動漫、油畫、素描等。

「日式水彩插畫風格，柔和暖色調」
🖼

以圖生圖

以上傳的圖片作為視覺參考，結合文字描述生成一張風格相近但全新的圖片。

「同樣的場景，但改為冬天雪景，冷藍色調」
✂️

元素提取

自動辨識並提取圖片中的主要人物或物件，將其放置到全新場景或背景中。

「將角色放置在宇宙星空的背景中，科幻氛圍」
🌄

背景替換

智慧偵測並保留前景主體，精確替換圖片背景為任意描述的場景。

「換成日落海邊，橙紅色天空與反光沙灘」

📋 使用步驟

1

切換到「圖片轉換」頁籤

在頂部導航列點擊「圖片轉換」（🖼），進入圖片轉換工作區。

2

上傳來源圖片

點擊上傳區或將圖片拖曳到虛線框內。支援 JPG、PNG 格式，大小上限 10MB。圖片上傳後可即時預覽。

3

選擇轉換模式

從 4 個模式卡片中選擇您想要的效果。不同模式的 Prompt 輸入提示也會隨之改變。

🎨 風格轉換 🖼 以圖生圖 ✂️ 元素提取 🌄 背景替換
4

輸入描述（自訂 Prompt）

在文字框中描述您期望的效果。系統會依照選擇的模式提供對應的範例提示，可作為參考。

5

選擇 AI 模型與輸出比例

選擇想使用的 AI 模型（Gemini Imagen 或 GPT Image 2）以及輸出圖片的寬高比例（1:1、16:9、4:3、9:16）。

6

點擊「開始 AI 轉換」

系統開始進行 AI 圖片轉換。轉換期間可隨時點擊取消。完成後結果圖片會顯示在右側預覽區，可直接下載。轉換記錄也會自動存入歷史紀錄。

🤖 AI 模型比較

項目 Gemini Imagen GPT Image 2
呼叫方式 透過後端 Azure Function 前端直接呼叫 Azure AI Foundry
模型識別碼 gemini-imagen gpt-image-2
API 類型 文字 + 圖片 Prompt 輸入 images/edits（multipart/form-data）
所需環境變數 GOOGLE_API_KEY（後端） VITE_GPT_IMAGE_EDIT_ENDPOINT（或自動推導）
需要前端 API Key ✗ 不需要 ✓ 需要（可選）
支援取消操作 ✓ ✓
輸出格式 Base64 data URL Base64 data URL 或圖片 URL
伺服器過載重試 ✓ 自動重試（最多 2 次） ✗ 單次請求

⚙️ 技術架構

// ─── 呼叫流程 ─────────────────────────────────────── 使用者上傳圖片 ↓ requestBlobSas() // 取得 Azure Blob SAS token ↓ 上傳至 Azure Blob Storage 選擇模式 + 輸入 Prompt ↓ aiService.transformImage() // ─── 依 model 分岔 ───────────────────────────────── if (model === "gemini-imagen") { → POST /api/image-transform // Azure Function → Gemini generateContent(image + prompt) → 回傳 base64 imageUrl } if (model === "gpt-image-2") { → editImageGpt() // 前端直接呼叫 → POST {GPT_IMAGE_EDIT_ENDPOINT} multipart: { image, prompt, size } → 回傳 base64 imageUrl } 結果顯示於右側預覽區 ↓ saveHistoryItem() // 自動存入歷史紀錄

💡 各模式的 AI Prompt 策略（後端自動組裝）

🎨 風格轉換
Transform the visual style of this image. Apply the following style: {使用者 Prompt}. Keep the same subjects, composition, and content, but change the color palette, artistic treatment, and visual style completely.
🖼 以圖生圖
Using this image as a visual reference and inspiration, generate a completely new image based on the following description: {使用者 Prompt}. Incorporate similar color palette, mood, and composition style.
✂️ 元素提取
Extract the main subjects and key elements from this image. Place them in a new scene or context as described: {使用者 Prompt}. Keep the extracted subjects recognizable but adapt them naturally to the new context.
🌄 背景替換
Keep the main foreground subjects and characters in this image exactly as they appear. Replace only the background with: {使用者 Prompt}. The foreground subjects must remain unchanged.

🔑 環境變數設定

⚠️
GPT Image 2 圖片編輯需要設定編輯端點。若已設定生成端點（VITE_GPT_IMAGE_ENDPOINT），系統會自動推導編輯端點（將 /images/generations 替換為 /images/edits）。
環境變數 位置 說明 必要性
GOOGLE_API_KEY 後端（Azure Functions 設定） Gemini API 金鑰，Gemini 模式必須 必要（Gemini）
VITE_GPT_IMAGE_ENDPOINT 前端（GitHub Secrets） GPT-Image-2 生成端點，已存在 已有
VITE_GPT_IMAGE_EDIT_ENDPOINT 前端（GitHub Secrets） GPT-Image-2 編輯端點，若不設定則自動推導 選填
VITE_GPT_IMAGE_API_KEY 前端（GitHub Secrets） GPT-Image-2 API 金鑰，已存在 已有
💡
修改 VITE_* 環境變數後，必須重新觸發 CI/CD（重跑 GitHub Actions workflow）才能生效，因為這些變數在 build 時注入，不是 runtime。
